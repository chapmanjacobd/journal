I made a little rust experiment of library fsadd --fs and published it as library-rs. I still don't really know what I'm doing with in Rust so a lot of it is vibe-coded. I spent a lot of time playing around with compiling libmagic on Windows (via GitHub Actions--I don't have access to a Windows machine). After trying both msys2 and a vcpkg approach for a few hours, not too mention that multi-threading with libmagic is a bit fiddly, I gave up and went with tree-magic-mini which works but it has some shortcomings.

I tried a bunch of different concurrency approaches via ChatGPT and Gemini but in the end the program is not quite as fast as the Python version which was somewhat surprising. For disks with a lot of really small files library-rs was significantly faster but most of the time it is a pretty close tie.
