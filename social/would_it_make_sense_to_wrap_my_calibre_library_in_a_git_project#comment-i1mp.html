I've done worse things with git... The only thing you really need to be careful of is to make sure that you aren't committing an open database. One thing that can help is to run

PRAGMA wal_checkpoint(TRUNCATE);

before git add file.db; git commit.

You can add this line to ~/.config/git/attributes to have git diff between sqlite changes:

*.db       diff=sqlite3

You could also add diff zip files. For example, listing zip contents:

git config --global diff.lsar_list.textconv lsar

tee -a ~/.config/git/attributes
*.zip diff=lsar_list
*.tar.gz diff=lsar_list
*.tgz diff=lsar_list
*.7z diff=lsar_list

or even file contents of files within zip and other supported archives... assuming the files within are text encoded:

git config --global diff.zip_content.command "pkgdiff"

tee -a ~/.config/git/attributes
*.zip diff=zip_content

But yeah storing large files in git will usually result in the git index being at least as big as the files you store so you lose 1/2 of your storage space
