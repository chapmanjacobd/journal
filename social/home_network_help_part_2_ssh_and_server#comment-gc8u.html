remoting into a server if you actually need the desktop

NoMachine works pretty well on low-bandwidth links. Getting NoMachine to run securely without paying money is more involved so I won't get into it here.

freerdp is also quite good! (but for some reason on my setup audio isn't forwarded). To use freerdp I run rdpserve.fish on the server and rdp.fish on the client in one go like this:

ssh -t pakon setsid -f 'fish -c "rdpserve"'
rdp pakon.my_tailscale_subdomain.ts.net:35589

All that being said I usually just use ssh instead of GUI because there is still latency with remote X11. The feeling of mstsc.exe connecting between two Windows machines is a lot better: the UI feels very responsive and clear.

If you're on Wayland maybe the situation is different with waypipe, etc.
